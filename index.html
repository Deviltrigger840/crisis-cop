CrisisCop: Intelligent Security Patrol Robot

The research endeavors to develop an effective and scalable patrol robot system that is capable of dealing with dynamic settings with enhanced adaptability and wisdom. It combines path adjustment through reinforcement learning-based method, LIDAR-SLAM for avoiding obstacles, and GPS-Waypoint navigation for adaptive traversing.

The system further suggests a sensor-based approach for adaptive patrol routes, predicted weather conditions, air quality, and body health parameters. The system accommodates seamless collaboration between slave robots using real-time communication and decision-making operations, assigning them in accordance with location, battery, and environmental parameters.

Real-time decision-making, proactive response, and optimal utilization of resources are ensured using Edge Computing, IoT, and Machine Learning (ML) techniques.

Research Framework

Literature Survey

This project enhances slave robots' adaptability by integrating external environmental data (e.g., gas, humidity, pressure) with internal health metrics (e.g., voltage, temperature) for proactive maintenance and real-time response. Unlike past studies, it introduces short-term environmental forecasting and dynamic power optimization based on environmental stressors. Combining sensor fusion with traction control and alert systems, it creates a holistic, adaptive robot system for resilient operation in diverse and changing warehouse conditions.

Overview

Aimed to design an autonomous outdoor patrol robot that can travel in the diverse terrain and weather conditions while being very reliable. In operation, real-time environmental analysis has adapted to these physical systems. These are designed for various groups including security teams, facility managers, or organizations seeking scalable surveillance solutions. The challenges in multi-sensor fusion algorithms involve the handling of missing and noisy data. It implements a strong algorithm that can make meaningful insights even from incomplete or unreliable data, hence enhancing the ability of the robots to make accurate environmental predictions for optimum performance. For now, the functionalities are being simulated with python.

Research Gap

Problem Statement

Traditional patrol systems struggle with adaptability, decision-making, and coordination in dynamic environments. This causes inefficiencies in navigation, monitoring, and resource use. A scalable, intelligent robot system is needed—capable of responding to real-time changes in environment and robot health—through integrated sensing, machine learning, and advanced communication technologies.

Research Objective

  • Develop an energy-efficient algorithm that minimizes power consumption while ensuring real-time monitoring and secure data transmission through edge computing.
  • Design a robotic system with Lidar, and GPS that patrols the warehouses autonomously. The route should automatically be updated according to environmental conditions.
  • A sensor-based system for slave robots enabling adaptive weather and AQI prediction, fault detection, and traction control. It uses real-time IoT data and a web dashboard to ensure reliable performance and autonomous decision-making.
  • An algorithm enabling real-time communication and coordination among robots, reducing latency and response time. It also autonomously detects incidents and dynamically adjusts patrol routes for enhanced coverage.

Methodology

The system combines Reinforcement Learning for adaptive path planning, LIDAR-SLAM for real-time mapping and obstacle avoidance, and GPS-waypoint navigation for precise routing. Onboard sensors monitor environmental and health data, processed via machine learning for predictive insights. Edge computing and IoT enable real-time communication, while a distributed coordination system ensures efficient, autonomous collaboration among slave robots.

Milestones

Milestone 1: Requirement Analysis and System Design

Define project goals and key use cases. Select hardware like master/slave robots, sensors, and software tools such as ROS and TensorFlow. Design the system architecture integrating edge computing, IoT, and machine learning.

Timeline: Week 1–2

Milestone 2: Hardware Integration and Sensor Setup

Assemble the robot units and install essential sensors including weather, gas, and health monitors. Test GPS and LIDAR modules to ensure basic navigation capabilities.

Timeline: Week 3–4

Milestone 3: Environment Data Collection and Preprocessing

Collect sensor data during patrols and clean it by handling noise and missing values. Prepare labeled datasets to support supervised learning models.

Timeline: Week 5–6

Milestone 4: Machine Learning Model Development

Train and optimize machine learning models for predicting weather and air quality. Validate their accuracy and deploy them on edge devices for real-time inference.

Timeline: Week 7–8

Milestone 5: Navigation and Obstacle Avoidance Implementation

Implement GPS waypoint navigation and integrate LIDAR-SLAM for real-time obstacle detection. Develop reinforcement learning algorithms to enable adaptive path planning.

Timeline: Week 9–10

Milestone 6: Slave Robot Coordination and Communication

Create protocols for real-time inter-robot communication. Implement autonomous decision-making based on battery status, location, and environmental inputs to ensure efficient collaboration.

Timeline: Week 11–12

Milestone 7: System Integration and Testing

Combine all modules into a cohesive system. Perform simulations and field tests in varied conditions to identify and fix issues for optimal performance.

Timeline: Week 13–14

Milestone 8: Final Review and Documentation

Prepare comprehensive documentation and final project report. Develop a demo video or presentation. Review project outcomes and measure success against initial goals.

Timeline: Week 15
